# 工作原理

paste 的核心只有三件事:把 URL 路径映射到一个 txt 文件、在浏览器里静默自动保存、以及在中间做好安全与数据完整性。下面逐一说明。

## 路径映射

访问 `/name` 时,服务把它映射到 `NOTES_DIR` 下的 `name.txt`:

```
/note   ⟷   notes/note.txt
/todo   ⟷   notes/todo.txt
```

- `GET /name` — 读取对应 txt 的内容,填进全屏文本框返回。文件不存在时返回空文本框,**第一次保存时才创建文件**。
- `POST /name` — 把请求体写回对应 txt。
- `GET /` — 重定向到 `/note`,作为默认入口。

映射由 `safe_path()` 完成:它校验名字合法后,拼出 `NOTES_DIR/name.txt` 的绝对路径。

## 静默自动保存

页面上没有任何按钮,保存完全自动。前端脚本这样兜底:

- **输入防抖**:每次 `input` 后重置计时器,停止输入约 600ms 才发一次 `POST`,避免每敲一个字都请求。
- **请求串行化**:保存请求不并发。若上一次还没返回,新的改动标记为 pending,等前一次完成再补发,避免旧内容覆盖新内容。
- **多重兜底保存**:文本框失焦(`blur`)、页面切到后台(`visibilitychange` 且 `document.hidden`)时都会触发保存。
- **离开页面**:`beforeunload` 时若还有未保存改动,用 `navigator.sendBeacon` 发出最后一次保存 —— 这种请求即使页面正在关闭也能可靠送达。

效果就是:你只管打字,内容会在停顿、切走、关页面时悄悄存好,不需要记得点保存。

## 安全与数据完整性

服务直接把 URL 名字拼进文件路径,所以要防路径穿越;写文件也要防并发损坏。实现上做了几层保护:

- **名字白名单**:合法名字必须匹配 `^[A-Za-z0-9][A-Za-z0-9._-]*$` —— 只允许字母数字和 `.` `_` `-`,且不能以点开头(避免 `.hidden` 之类隐藏文件)。
- **拒绝穿越**:名字里含 `..` 直接拒绝。
- **双重保险**:拼出绝对路径后,再用 `os.path.commonpath` 确认它仍在 `NOTES_DIR` 之内,否则拒绝。任何一步不合法都返回 `400`。
- **原子写入**:保存时先写 `name.txt.tmp`,再 `os.replace` 原子重命名成 `name.txt`。这样并发写或中途中断都不会写出半截损坏的文件。

服务用 `ThreadingHTTPServer`,能同时处理多个连接;这也是为什么写入要做原子替换。
